## 1️⃣ CPR - 응급 처치의 핵심 용어

**CPR**은 "Cardiopulmonary Resuscitation"의 약자로, 한국어로는 **심폐소생술**이라고 부릅니다. 이것은 심장이 멈추거나 호흡이 중단된 사람을 살리기 위한 응급 처치 방법입니다.

힌트에서 "Pink Pony Club"의 비트에 맞춰 한다고 한 이유가 흥미로운데, 이 곡의 템포(약 128 BPM)가 CPR을 할 때 필요한 가슴 압박의 속도와 정확히 일치하기 때문입니다. 응급 상황에서 효과적인 CPR을 하려면 분당 100~120회의 압박이 필요한데, 이 곡을 부르거나 머릿속으로 생각하면서 압박하면 정확한 속도를 유지할 수 있다는 것이죠!

**예문**: "Everyone should learn CPR because you never know when you might need to save someone's life." (모든 사람이 CPR을 배워야 합니다. 왜냐하면 언제 누군가의 생명을 구해야 할지 모르기 때문입니다.)

## 2️⃣ OHOK - 캐주얼한 동의 표현

**OHOK**는 "Oh OK"를 줄인 인터넷 슬랭으로, **"아, 알겠어"** 또는 **"오, 그렇구나"**라는 뜻입니다. 힌트의 "Ahh, gotcha gotcha"는 상대방의 말을 이해했다는 의미의 캐주얼한 반응을 나타냅니다.

이것은 주로 문자 메시지나 SNS에서 사용되는 매우 비공식적인 표현입니다. 실제 회화에서는 "Oh, I see," "Got it," 또는 "I understand"처럼 더 정중한 표현을 사용하는 것이 좋습니다.

**예문**:

- 친구: "I'm running 20 minutes late for our meeting."

- 당신: "OHOK, no problem. See you soon!" (오, 알겠어. 괜찮아. 곧 봐!)

## 3️⃣ BASIC - 컴퓨터 프로그래밍의 역사

**BASIC**은 "Beginner's All-purpose Symbolic Instruction Code"의 약자로, **초보자를 위한 프로그래밍 언어**입니다. 1960년대에 개발되어 개인용 컴퓨터 초기 시대에 매우 인기 있었던 언어입니다.

"Old-school"이라는 힌트처럼, BASIC은 이제 거의 사용되지 않는 오래된 언어이지만, 프로그래밍의 기초를 배우기에는 여전히 좋은 언어로 평가받고 있습니다. 요즘 초보자들은 Python이나 JavaScript 같은 현대적인 언어를 배우지만, 컴퓨터 역사를 이해하려면 BASIC을 알아두는 것도 좋습니다.

**예문**: "My grandfather learned to code using BASIC back in the 1980s." (할아버지는 1980년대에 BASIC을 사용해서 코딩을 배웠어요.)

## 4️⃣ WEBSITE - 웹 브라우징의 기본 개념

**WEBSITE**는 **웹사이트**, 즉 **인터넷에서 볼 수 있는 웹 페이지들의 모음**을 의미합니다. 힌트의 "Browser destination built by a programming language"는 웹사이트가 HTML, CSS, JavaScript 같은 프로그래밍 언어로 만들어진다는 점을 설명하고 있습니다.

WEBSITE는 "WEB"(웹)과 "SITE"(장소, 위치)의 합성어입니다. 우리가 매일 방문하는 Google, YouTube, Netflix 등이 모두 웹사이트입니다.

**관련 어휘**:

- **Web browser**: 웹사이트를 보기 위한 프로그램 (Chrome, Safari, Firefox 등)

- **Web page**: 웹사이트를 구성하는 개별 페이지

- **URL**: 웹사이트의 주소 (예: www.example.com)



**예문**: "I found a really useful website that helps you learn English vocabulary." (영어 어휘를 배우는 데 도움이 되는 정말 유용한 웹사이트를 찾았어요.)

## 5️⃣ SEETO - 탑승의 다양한 표현

**SEETO**는 **"see to"라는 동사구의 의미**를 담고 있습니다. 이것은 **"~을 돌보다," "~을 처리하다," 또는 "~을 확인하다"**라는 뜻입니다. 그런데 여기서 "Get on"의 의미로는 **"탑승하다"** 또는 **"타다"**라는 뜻으로 사용되고 있습니다.

사실 "Get on"은 여러 의미가 있습니다:

- **탈것을 타다**: "Get on the bus" (버스를 타다)

- **잘 지내다**: "How are you getting on?" (어떻게 지내세요?)

- **진행되다**: "How is the project getting on?" (프로젝트가 어떻게 진행 중인가요?)



**예문**:

- "I need to get on the train in five minutes." (5분 안에 기차를 타야 해요.)

- "She got on her bicycle and rode away." (그녀는 자전거를 타고 떠났어요.)
